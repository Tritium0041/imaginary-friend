# 《时空拍卖行》桌游 Agent 系统产品需求文档 (PRD)

## 1. 产品概述

本产品旨在为开源桌游《时空拍卖行》（Chronos Auction House）开发一套支持单人游玩的 Agent 系统。玩家作为唯一的真实人类玩家，将与多个由大语言模型驱动的 AI Agent 共同进行游戏。系统不仅负责模拟其他玩家的决策与交互，还包含一个独立的 GM（Game Master）Agent 负责规则裁定、状态管理与黑盒信息处理。该系统通过自然语言交互和自动化的规则执行，为玩家提供沉浸式的单人桌游体验。

## 2. 核心架构与角色定义

系统采用去中心化与中心化结合的架构，主要包含真实玩家、玩家 Agent 和 GM Agent 三个核心实体。

真实玩家是系统的唯一真实用户，通过 Web UI 或 GUI 与系统进行文字交互。在轮到自己行动时，真实玩家需要做出决策，并在规则出现歧义时拥有最终裁定权。

玩家 Agent 充当游戏中的其他对手，支持 3-5 人局，即 2-4 个 Agent。它们具备独立的思考能力、局内记忆与性格特征，能够通过自然语言与其他玩家（包括真实玩家）进行对话，例如交易谈判或拍卖放话。Agent 的思考过程对真实玩家不可见，即处于黑盒状态，前端界面仅输出最终的行动结果与对话内容。

GM Agent 是独立于玩家的系统级 Agent，负责维护全局游戏状态，如当前回合、稳定性、各时代倍率和公共拍卖区等。它处理需要信息隔离的机制，例如密封竞标的统一收集与揭示。在规则执行出现歧义时，GM Agent 会暂停游戏并向真实玩家提问，由玩家进行最终拍板。

## 3. Agent 系统设计

### 3.1 身份与性格库

系统内置一个轻量级的身份库，每局游戏开始时为每个 Agent 随机抽取身份。身份设定决定了 Agent 在游戏中的行为模式和交互风格。

| 身份属性 | 描述说明 | 示例 |
| --- | --- | --- |
| **名称** | Agent 的称呼，体现其角色定位 | 激进的投机客、保守的收藏家 |
| **说话风格** | 影响其在游戏中的对话语气和用词 | 挑衅、谨慎、圆滑 |
| **策略倾向** | 作为 Agent 的固定标签，指导其核心决策 | 偏好收集特定时代文物、喜欢使用功能卡破坏他人、热衷于操纵倍率 |

Agent 在游戏过程中会以该策略倾向为基础，根据当前局势（如稳定性危机、倍率变化）进行小幅度的动态调整，以保证游戏的挑战性和真实感。

### 3.2 记忆与思考机制

Agent 的记忆系统专注于当前单局游戏，不跨局继承信息。它们可以随时调用系统提供的状态查询工具获取当前的公开游戏状态。基于这些公开信息，Agent 能够推断其他玩家的手牌和策略意图，例如通过观察对手的出价高低或投票方向来猜测其目标。Agent 的推理过程在后台进行，前端界面仅展示其最终的决策动作和伴随的对话，确保玩家体验的沉浸感。

## 4. 游戏流程与交互机制

### 4.1 交互节奏

游戏流程由 GM Agent 自动推进。当轮到玩家 Agent 行动时，它们会自动执行决策并输出结果。当轮到真实玩家行动时，系统会暂停并等待玩家输入决策。这种回合制的交互节奏保证了游戏的有序进行。

### 4.2 核心机制的 Tool Call 实现

为了保证游戏规则的严谨性和信息隔离，系统通过特定的 Tool Call 实现关键机制。

| 机制名称 | 实现方式 | 作用说明 |
| --- | --- | --- |
| **状态查询** | 提供全局 Tool，所有实体均可调用 | 获取当前回合数、稳定性、倍率、各玩家公开资产等信息 |
| **密封竞标** | 参与者将出价通过 Tool 发送给 GM | GM 收集完毕后统一进行黑盒揭示，判定胜负并公布结果，彻底解决纯文字交互中 Agent 可能“偷看”玩家出价的问题 |

### 4.3 规则裁定与异常处理

当游戏过程中出现规则边缘情况或功能卡效果冲突时，GM Agent 会识别出歧义并暂停游戏推进。GM 会向真实玩家清晰描述冲突点，并提供可能的判定选项。真实玩家做出最终裁定后，GM 依据裁定结果继续推进游戏。这种机制确保了复杂规则下的游戏能够顺利进行，同时赋予了玩家掌控感。

## 5. 界面与技术形态

系统初期优先实现可运行的核心逻辑版本，并套用简单的 Web UI 或 GUI。交互方式以纯文字对话和指令为主，玩家可以通过自然语言输入行动，如“我出价 10”或“我用这张卡换你的那张”，系统会自动解析并执行。界面设计需明确区分系统提示、Agent 行动结果与 Agent 角色对话，以增强游戏的沉浸感和可读性。
